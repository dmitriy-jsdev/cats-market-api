# Cats Market API — Node.js + TypeScript + Express

Небольшой REST API для интернет-магазина кошек 🐱: отдаёт список «товаров» (котов) с пагинацией, раздаёт изображения и предоставляет эндпоинты для регистрации и входа пользователя. Предполагается использование в паре с фронтендом **Cats Market** (отдельный репозиторий).

---

## ✨ Возможности

- **Список котов**

  - `GET /api/products` — возвращает список котов с пагинацией.
  - Параметры запроса:
    - `page` — номер страницы (по умолчанию `1`);
    - `limit` — количество элементов на странице (по умолчанию `5`).
  - Ответ содержит:
    - `products` — массив котов;
    - `pages` — общее количество страниц.

- **Авторизация**

  - `POST /api/sign_up` — регистрация нового пользователя.
    - Валидация логина и пароля (обязательность, длина).
    - Проверка на уникальность имени пользователя.
    - Пароли хранятся в виде хэшей (`bcryptjs`).
  - `POST /api/sign_in` — вход по логину и паролю.
    - Проверка существования пользователя.
    - Сравнение пароля с хэшем.
    - При успехе генерируется JWT и сохраняется в httpOnly cookie `token`.
  - `GET /api/me` — получение текущего пользователя по cookie `token`.
    - При отсутствии/невалидности токена возвращает `401`.

- **Статические файлы**

  - Раздача изображений котов из `public/images/cats` по пути:
    - `GET /images/cats/<имя_файла>`

- **CORS и JSON**
  - Включён CORS для запросов с фронта.
  - Поддержка JSON-тела запросов (`express.json()`).

## 🧰 Стек

- Node.js, TypeScript
- Express
- CORS
- Хэширование паролей: `bcryptjs`

- Сборка: `typescript` + `tsc`
- Dev-режим: `ts-node-dev`

## 📦 Данные

- Коты описаны как мок-данные в TypeScript-модуле:

  - `src/data/cats.ts` — интерфейс `Product` и массив `ALL_CATS`.
  - Для картинок используется базовый URL, настраиваемый через переменную окружения:
    - `BASE_IMAGE_URL` (по умолчанию `http://localhost:3000/images/cats`).

- Пользователи:
  - Хранятся **в памяти процесса** (in-memory) в модуле:
    - `src/data/users.ts`
  - Данные предназначены только для демо: **при перезапуске сервера пользователи сбрасываются**.
  - Демо-пользователь по умолчанию:
    - `username`: `Dmitry`
    - `password`: `11111111`

## 🔐 Авторизация и токены

- При `POST /api/sign_in`:

  - Генерируется JWT вида `{ userId, username }`.
  - Токен подписывается секретом `JWT_SECRET`.
  - Время жизни задаётся константой (`7d` по умолчанию).
  - Токен отправляется клиенту в виде httpOnly cookie `token`.

- Переменные окружения:
  - `PORT` — порт HTTP-сервера (по умолчанию `3000`).
  - `JWT_SECRET` — секрет для подписи JWT (по умолчанию dev-значение, в проде обязательно переопределить).
  - `BASE_IMAGE_URL` — базовый URL для формирования путей к изображениям котов.
